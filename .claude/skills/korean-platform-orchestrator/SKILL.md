# 국어 플랫폼 오케스트레이션

## Phase 0: 컨텍스트 확인

`source/_posts/`, `DESIGN.md`, `_workspace/` 존재 여부를 확인한다. 기존 산출물이 있고 부분 수정 요청이면 해당 레슨·레이아웃만 수정한다. 새 입력으로 전체 방향이 바뀌면 기존 중간 산출물은 `_workspace_prev/`로 보존한다.

한글 문학 자료가 입력되면 `korean-literature-source-parsing`을 먼저 사용한다. HWP·HWPX·PDF·압축 파일의 문자 인코딩과 추출 결과를 확인한 뒤, 원문과 해설은 임시 분석 범위에만 두고 식별 메타정보만 콘텐츠 단계로 전달한다.

## Phase 1: 구조 확인

초등, 중등, 고등 2028 경로와 각 레슨의 front matter를 확인한다. 고등은 화법과 언어, 독서와 작문, 문학 공통 구조만 허용한다.

## Phase 2: 콘텐츠와 QA

`content-curator`는 학습 글을 만들고, `curriculum-qa`는 학년 적합성과 사실성을 검토한다. 권리 불명확 원문은 자체 해설로 대체한다.

## Phase 3: 게시 검증

`site-maintainer`는 `npm test`와 `npm run build`를 실행하고 생성 URL·링크·필터 버튼의 접근성 상태를 점검한다. 한 번 재시도 후 같은 오류가 나면 누락을 명시하고 중단한다.

## 데이터 전달

작업 상태는 파일 경로와 Git diff로 전달한다. 큰 조사 자료는 `_workspace/{phase}_{agent}_{artifact}.md`에 보존한다.

## 테스트 시나리오

- 정상: “중등 문법 레슨만 추가”는 기존 디자인과 허브 분류를 유지한 채 새 글과 생성 페이지를 검증한다.
- 오류: 고등 레슨이 이전 선택과목 구조로 분류되면 QA가 수정 요청을 내고 빌드 전에 차단한다.
