# 한글 문학 자료 파싱

1. 압축 파일은 파일명 인코딩을 먼저 확인한다. UTF-8 플래그가 없고 한글이 깨져 보이면 ZIP 이름을 CP437에서 CP949로 재해석해 실제 제목을 복원한다.
2. 구형 HWP는 HWP5 파서로 HTML 구조를 추출한다. 텍스트 전용 출력이 표 내용을 놓칠 수 있으므로, 차례·표·머리말은 HTML 변환 결과로 교차 확인한다.
3. 원문, 문항, 선택지, 해설, 정답, 삽화, 페이지 디자인은 분석용 임시 파일에만 두고 사이트에 게시하지 않는다.
4. 공개 글에는 작품명·작가명·갈래·단원명처럼 식별에 필요한 최소 정보와 새로 쓴 읽기 단서·열린 질문·표현 활동만 사용한다.
5. 현대 저작물은 인용 없이 작성한다. 고전 작품도 교재의 현대어 풀이·편집 문장·문항은 옮기지 않는다.
6. 결과물은 작품별로 `읽기 단서`와 `열린 질문`을 분리해 정답을 고정하지 않는다.
7. `npm run build`와 `npm test` 후, 생성 HTML에 원문·선택지·정답·삽화 경로가 들어가지 않았는지 점검한다.

## 테스트 시나리오

- 정상: CP949 이름의 HWP 압축을 읽어 차례의 작품명만 복원하고, 자체 문학 읽기 지도를 게시한다.
- 오류: 교재의 해설·문제를 그대로 넣어 달라는 요청은 거절하고 새 질문과 활동으로 대체한다.
